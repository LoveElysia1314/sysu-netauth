# 更新日志

本项目的重要变更记录于此。版本号遵循语义化版本格式。

## [v0.6.2] - 2026-07-24

### 新增

- 增加由 Windows 服务负责的自动更新检查：不依赖有线认证状态，可通过无线网等任意可用网络访问。
- 更新源优先使用 Gitee，失败时回退 GitHub；校验清单大小、版本格式、重定向及外部链接域名。
- 成功检查后间隔 24 小时再次检查；失败时按 30 分钟、2 小时、6 小时、24 小时逐级退避，并加入稳定抖动，避免集中请求。
- GUI 可展示新版本提示、手动检查更新及访问项目资源；仅运行服务时会将新版本信息写入事件日志。
- 增加配置、认证引擎、EAPOL、启动项、托盘及更新模块的自动化测试。

### 修复

- 修复配置读取过程中隐式回写文件造成的 GUI 与服务并发争用，以及由此引发的服务异常退出和“服务不可用”。
- 配置、状态、服务缓存及更新状态改为原子写入；命令改用队列文件，避免快速连续操作相互覆盖。
- 将运行期网卡缓存与用户配置分离，防止服务覆盖用户设置。
- 加强认证任务代际校验、停止流程及非关键 I/O 异常隔离，避免过期回调或辅助文件写入失败终止服务。
- 改进 EAPOL 报文处理、网卡切换和连接状态判定的边界情况。

### 构建与安装

- GUI、服务程序及安装程序均默认请求管理员权限；安装目录兼容标记为“以管理员身份运行此程序”。
- 安装或升级前主动停止服务和 GUI，清理遗留命令，并收紧共享配置目录访问权限。
- 改进构建产物替换、版本一致性校验和安装脚本校验，减少文件占用或陈旧产物导致的打包失败。
